Project 1: Server-side API Design using Node.js, Express.js, MongoDB, and Mongoose

Use Postman or another program to test API routes.

Description of API design: This API includes a list of users, which each have a unique username and email associated with them, as well as names, and can be accessed through /users. Recipes are under a separate route at /recipes, and contain descriptions, ingredients, instructions, and times for prep and cooking. They also have a list of reviews associated with them. Reviews are a separate resource within the recipe model, and are accessed by using /recipes/reviews to load all reviews or /recipes/recipeid/reviews/reviewid to access a specific review. Visit users/index.js and recipes/index.js for more specific description of routes, and their respective .model.js files for info on schema used.
Extra Credit: added GUI to view and navigate contents of users, recipes, and reviews using AngularJS. Implemented links between recipes and their reviews, and reviews and the user that created them, using the objectid framework used in the API.
